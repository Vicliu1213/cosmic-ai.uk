【新增目錄功能說明】
════════════════════════════════════════════════════════════════════════════

1. environments/
   用途: 存儲不同環境（開發、預發佈、生產）的配置
   內容: 每個環境有獨立的 config.json 和 README 說明
   好處: 快速切換不同環境配置
   
2. examples/
   用途: 提供配置示例和遷移指南
   內容: complete(完整示例)、custom(自定義示例)、migration(遷移指南)
   好處: 幫助用戶快速上手和遷移
   
3. backups/
   用途: 存儲配置文件備份
   結構: daily(日備份)、weekly(週備份)、monthly(月備份)
   好處: 方便版本控制和災難恢復

4. schemas/
   用途: 配置驗證模式定義
   內容: API、Engine、System、Trading 四個模式
   好處: 確保配置文件格式和內容正確
