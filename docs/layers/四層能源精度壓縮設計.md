# 四層能源精度壓縮計算設計

> 目標：以低 token、可分布式、可配置的方式，建立四層能源精度壓縮流水線。

## 四層定義
1. energy-observation：收集系統基線資料
2. energy-normalization：正規化與去噪
3. energy-compression：壓縮與 token 裁切
4. energy-distribution：分布式動態縮放與重新平衡

## 已補齊資料類
- LayerConfig
- CompressionConfig
- DistributedConfig
- LayerSystemConfig
- EnergySample
- EnergyTrace
- PrecisionLayerResult
- EnergyTelemetry
- EnergyPrecisionCompressionPipeline

## 四個資料來源
- runtime telemetry
- scheduler pressure
- precision-quality signals
- distributed locality signals

## 驗證方式
- `python - <<'PY' ...` 匯入 `src.layers`
- 執行 `build_default_pipeline().run()`
- 檢查輸出包含 4 層結果、4 個 trace、config 與 final 摘要
