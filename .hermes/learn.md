# Learning Loop

## Purpose
Capture durable improvements discovered during work.

## Learning policy
- If a workflow repeats, convert it into a skill.
- If a repository convention emerges, write it into docs.
- If a bug reveals a stable pattern, capture the fix in tests.
- If the user preference is stable, store it in memory.

## What to learn
- Reliable commands
- File layout conventions
- Dashboard interaction patterns
- Agent control patterns
- Verification habits

## Promotion rules
- Temporary findings stay in session notes.
- Stable workflows become skills.
- Stable preferences become memory.
- Stable project rules become files.
