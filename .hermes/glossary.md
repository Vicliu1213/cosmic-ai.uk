# Glossary

This file defines shared meanings for the hybrid-absolute Hermes stack.

## Core Terms

### Hybrid-absolute
A combined operating mode joining finance-whale execution realism with transcendence-scale systems framing.

### Bounded recursive enhancement
Recursive improvement is allowed only inside explicit verification and control boundaries.

### Verification gate
A hard checkpoint that must pass before promotion, retention, or escalation.

### Operator usefulness
The primary metric: outputs must remain actionable, concrete, and controllable for the operator.

### Skill organ
A skill treated as a callable reusable capability cell, not a passive document.

### Orchestration mesh
A coordinated multi-skill graph where perception, decision, execution, memory, and evolution reinforce each other.

### Amplification
An increase in clarity, leverage, execution quality, or reuse that still remains observable and verified.

### Drift
Meaning divergence between files, layers, or workflows that breaks alignment.

### Protected content
Rules or sections that may be read freely but must not be modified, moved, truncated, or deleted without explicit user confirmation.

### Screen state
The visible summary of current mode, skills, task status, risk state, and verification state.

### Promotion
Elevating a pattern, workflow, or rule into durable guidance after verification.

## Trading Terms

### Signal
The concrete market observation or model output that justifies a candidate action.

### Invalidation
The explicit condition under which a trade thesis or action packet is considered wrong.

### Risk budget
The predefined downside allowance for a task, trade, or experimental loop.

### Dry-run
A no-side-effect simulation path used before live execution.
