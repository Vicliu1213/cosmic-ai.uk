## 使用方法

### 方式 1: 直接運行腳本（推薦）

```bash
cd /root/comic_ai
./activation_status.sh
```

### 方式 2: 直接調用 Python

```bash
cd /root/comic_ai
source venv/bin/activate
python activation_status_cli.py
```

### 方式 3: 從任何位置

```bash
bash /root/comic_ai/activation_status.sh
```

## 主菜單選項

進入儀表板後，您可以選擇以下操作：

### [1] 運行測試套件
執行完整的 Comic AI 測試套件（218 項測試）。顯示詳細的測試結果和統計信息。

```bash
pytest src/tests/ -v
```

**預期輸出**:
- 218 通過
- 0 失敗
- 運行時間: ~1.3 秒

### [2] 啟動最小演示
運行 `demo_complete_system.py`，展示所有 7 個核心功能：

1. 文件處理系統
2. 多智能體交易
3. 量子優化（Grover 算法）
4. 模型推理
5. 性能監控
6. 日誌管理
7. 工作流集成

**預期輸出**:
- 中文界面完整演示
- 所有功能正常工作
- 運行時間: ~3.5 秒

### [3] 啟動所有應用
使用 TMUX 同時啟動所有 7 個應用程序。

應用程序列表：
- 文件處理 CLI
- 日誌儀表板 (Port 5000)
- 任務面板 (Port 5001)
- 混合雲儀表板 (Port 5002)
- 多智能體交易演示
- Gemini 交易分析師
- 主 CLI 介面

**加入 TMUX 會話**:
```bash
tmux attach-session -t comic-ai-apps
```

### [4] 查看應用指南
顯示 `APPS_USAGE_GUIDE.md` 的內容，包含所有應用程序的詳細使用說明。

### [5] 提交並推送代碼
自動執行以下操作：
1. 添加新文件到 Git 暫存區
2. 創建提交
3. 推送到遠程倉庫

### [6] 刷新狀態
重新掃描系統狀態並刷新儀表板顯示。

### [q] 退出
退出激活狀態儀表板。

## 技術詳節

### 架構

激活狀態 CLI 由以下組件組成：

1. **ActivationStatusCLI 類**
   - 初始化和配置
   - 狀態掃描方法
   - 菜單管理
   - 命令執行

2. **狀態檢查方法**
   - `get_git_status()` - 檢查 Git 狀態
   - `get_test_status()` - 運行並檢查測試
   - `verify_applications()` - 驗證應用文件

3. **顯示方法**
   - `display_activation_status()` - 主儀表板
   - `print_header()` - 標題格式
   - `print_section()` - 分段標題

### 依賴項

- Python 3.12+
- subprocess（用於 Git 和 pytest 命令）
- pathlib（用於文件路徑）
- datetime（用於時間戳）

## 故障排除

### 問題: "虛擬環境未找到"

**解決方案**:
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 問題: "Python 命令未找到"

**解決方案**:
確保已正確激活虛擬環境：
```bash
source venv/bin/activate
which python
```

### 問題: "Git 狀態檢查失敗"

**解決方案**:
確保在 Git 倉庫目錄中：
```bash
cd /root/comic_ai
git status
```

## 更新和維護

### 更新激活狀態

如果添加了新的激活階段或步驟，編輯 `activation_status_cli.py` 中的 `self.activation_phases` 字典：

```python
self.activation_phases = {
    "新階段": {
        "狀態": "✅",
        "步驟": [
            ("步驟 1", "✅"),
            ("步驟 2", "✅"),
        ]
    }
}
```

### 添加新的菜單選項

在 `display_activation_status()` 方法的 `menu_options` 列表中添加新項目。
