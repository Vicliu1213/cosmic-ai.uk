# Unified Integrated Dashboard - User Guide

## 📊 概述

**統一整合儀表版** 是 Cosmic AI 系統的核心管理工具，將所有重要系統（進度追蹤、系統記憶、導覽索引、恢復系統、量子系統）整合在一個互動式界面中。

## 🚀 快速開始

### 方式 1: 直接運行 Python 腳本
```bash
cd /workspaces/cosmic-ai.uk
python3 system/dashboard/unified_dashboard.py
```

### 方式 2: 使用啟動腳本（推薦）
```bash
cd /workspaces/cosmic-ai.uk
bash system/dashboard/launch_unified_dashboard.sh
```

### 方式 3: 簡短命令（須添加別名）
```bash
# 在 ~/.bashrc 或 ~/.zshrc 中添加
alias cosmic-dash="python3 /workspaces/cosmic-ai.uk/system/dashboard/unified_dashboard.py"

# 然後直接執行
cosmic-dash
```

## 🎯 各視圖詳解

### [1] 📍 統一儀表版 (Unified Dashboard)
**用途**: 快速瀏覽所有系統的狀態

**顯示內容**:
- 🔹 **A 層 - 進度追蹤**: 當前進度的前 10 行
- 🔹 **B 層 - 系統記憶**: 系統記憶的前 10 行
- 🔹 **C 層 - 導覽索引**: 導覽索引的前 10 行
- 🔹 **系統狀態**: 恢復系統和量子系統的狀態
- 🔹 **信息**: 更新時間和工作目錄

**何時使用**:
- 每天開始工作時
- 想要快速了解系統狀態
- 系統健康檢查

### [2] 💾 A 層進度追蹤 (Progress Tracker)
**用途**: 查看詳細的進度信息

**顯示內容**:
- 當前工作流程
- 進行中的任務
- 完成情況
- 下一步計劃

**相關文件**: `system/tracking/PROGRESS_TRACKER.md`

**何時使用**:
- 需要了解當前工作進度
- 查看待辦事項
- 確認任務狀態

### [3] 📊 B 層系統記憶 (Memory)
**用途**: 訪問完整的系統激活記錄和歷史信息

**顯示內容**:
- 系統激活紀錄
- 過去的工作摘要
- 快速導航指南
- 關鍵決策記錄

**相關文件**: `memory.md`

**何時使用**:
- 需要查看完整歷史信息
- 尋找過去做過的相似工作
- 理解系統演進過程

### [4] 🗂️ C 層導覽索引 (Navigation Index)
**用途**: 快速導航到任何文件或資源

**顯示內容**:
- 完整的文件結構索引
- 快速開始指南
- 常見問題解答
- 重要鏈接

**相關文件**: `system/navigation/INDEX.md`

**何時使用**:
- 找不到某個文件
- 需要快速定位資源
- 新手入門

### [5] 🔍 組件狀態 (Component Status)
**用途**: 檢查所有系統組件的狀態和大小

**顯示內容**:
- 各組件加載狀態
- 文件大小
- 項目/行數
- 文件路徑

**組件清單**:
| 組件 | 狀態位置 | 用途 |
|------|--------|------|
| A 層 | system/tracking/ | 進度追蹤 |
| B 層 | / | 系統記憶 |
| C 層 | system/navigation/ | 導覽索引 |
| 恢復系統 | data/state/ | 恢復狀態 |
| 量子系統 | data/state/ | 量子狀態 |

**何時使用**:
- 診斷系統問題
- 確認所有組件存在
- 檢查文件大小

### [6] 🏥 健康檢查 (Health Check)
**用途**: 評估系統整體健康狀況

**顯示內容**:
- 詳細的檢查項目
- 健康評分（%）
- 通過/失敗統計
- 系統建議

**檢查項目**:
1. 系統文件夾
2. 數據文件夾
3. 進度追蹤文件
4. 系統記憶文件
5. 導覽索引文件
6. 恢復狀態文件
7. 量子狀態文件

**健康評分標準**:
- ✅ 100%: 系統完美
- ⚠️ 80-99%: 基本正常
- ❌ <80%: 需要維護

**何時使用**:
- 定期系統檢查
- 遇到問題時診斷
- 維護系統

### [7] 🔄 刷新 (Refresh)
**用途**: 重新加載所有數據

**何時使用**:
- 數據可能已更新
- 菜單顯示不完整
- 手動同步狀態

## 💡 使用技巧

### 快速巡視系統
1. 啟動儀表版
2. 按 [1] 查看統一儀表版
3. 按 [6] 進行健康檢查
4. 按 [0] 退出

### 深入了解進度
1. 按 [2] 查看 A 層進度
2. 按 [5] 查看組件狀態
3. 按 [3] 查看 B 層記憶了解背景

### 尋找文件或資源
1. 按 [4] 查看 C 層導覽索引
2. 使用 Ctrl+F 搜索關鍵詞
3. 查看文件路徑定位文件

### 故障診斷
1. 按 [6] 健康檢查
2. 按 [5] 查看組件狀態
3. 按 [1] 統一儀表版查看整體情況
4. 根據結果采取行動

## 🔧 故障排除

### 問題: "無法加載某個組件"
**原因**: 組件文件不存在或損壞

**解決方案**:
```bash
# 檢查文件是否存在
ls -la system/tracking/PROGRESS_TRACKER.md

# 查看詳細錯誤
python3 system/dashboard/unified_dashboard.py
```

### 問題: "健康評分低於 80%"
**原因**: 某些關鍵文件缺失

**解決方案**:
```bash
# 運行恢復系統
python3 system/recovery/cosmic_auto_recovery.py

# 驗證文件
bash system/dashboard/launch_unified_dashboard.sh
```

### 問題: "菜單無響應"
**原因**: 可能是文件太大或系統資源不足

**解決方案**:
```bash
# 強制退出
Ctrl+C

# 重新啟動
python3 system/dashboard/unified_dashboard.py
```
